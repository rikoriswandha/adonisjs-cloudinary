- **002, 003, 005 depend on 001**: until the lint errors are fixed, `npm run build`
  and `npm test` abort in their `pre*` hooks, so those plans cannot verify their own
  done criteria. Land 001 first.
- **003 depends on 002**: both touch `package.json` `exports` and `tsdown.entry`.
  Doing 002 first establishes the multi-entry build pattern that 003 extends; doing
  them out of order causes merge friction in the same two config blocks.
- **004 and 006 are independent** and can land any time (004 is still a release
  blocker; 006 is a docs fix).
- **007, 008, and 009 are independent** docs-only plans. They can land in any order
  and in parallel, but each executor should limit changes to its own in-scope files

## Verification gates (this repo)

| Purpose    | Command              | Expected on success            |
| ---------- | -------------------- | ------------------------------ |
| Typecheck  | `npm run typecheck`  | exit 0, no errors              |
| Lint       | `npm run lint`       | exit 0                         |
| Tests      | `npm run quick:test` | `56 passed` (more after plans) |
| Tests+gate | `npm test`           | runs lint then tests, exit 0   |
| Build      | `npm run build`      | exit 0, populates `build/`     |

Note: `npm test` runs `npm run lint` first (`pretest`), and `npm run build` runs
`npm run lint` first (`precompile`). A lint failure aborts both. `npm run quick:test`
bypasses the lint gate and is the fastest way to run only the test suite.

  Direction item (verify against live SDK / add integration coverage), not a
